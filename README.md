# 🧩 Практична задача

## **"Калькулятор особистого бюджету — рівень 1: місячний бюджет"**

---

## 1) 🎯 Навчальна мета

Після виконання завдання студент повинен:

* закріпити базові навички роботи зі Streamlit;
* вміти створювати інтерактивні форми введення даних;
* навчитися обробляти користувацький ввід;
* виконувати базові обчислення;
* відображати результати у вигляді таблиць і графіків;
* структурувати простий Python-проєкт.

---

## 2) 🧾 Сценарій / контекст

Ви створюєте простий веб-додаток для користувача, який хоче контролювати свої фінанси.

Користувач вводить:

* свій місячний дохід;
* витрати за категоріями (наприклад: їжа, транспорт, розваги тощо).

Додаток аналізує ці дані та показує фінансовий результат за місяць.

---

## 3) ⚙️ Функціональний опис

Студент має реалізувати Streamlit-додаток, який:

### Ввід даних:

* дозволяє ввести:

  * місячний дохід (`number_input`);
  * мінімум 3 категорії витрат;
  * суми витрат для кожної категорії;

### Обробка:

* обчислює:

  * загальні витрати;
  * залишок коштів (дохід - витрати);

### Логіка:

* визначає статус бюджету:

  * **профіцит** (залишок > 0)
  * **баланс** (залишок = 0)
  * **дефіцит** (залишок < 0)

### Вивід:

* відображає:

  * таблицю витрат;
  * загальні витрати;
  * залишок;
  * статус бюджету;
* будує простий графік (наприклад: bar chart або pie chart);

### (Опційно):

* зберігає результати у CSV-файл.

---

## 4) 🛠 Технічні вимоги

* Python 3.10+
* Streamlit
* pandas (рекомендовано для роботи з таблицями)

---

## 5) 📦 Обов’язкові компоненти Streamlit

Студент **обов’язково має використати**:

* `st.title()` або `st.header()`
* `st.number_input()`
* `st.text_input()` або фіксовані категорії
* `st.button()`
* `st.write()` / `st.dataframe()`
* один із графіків:

  * `st.bar_chart()` або
  * `st.line_chart()` або
  * `st.pyplot()` (опційно)

---

## 6) 🚧 Обмеження

* не використовувати складні UI-бібліотеки;
* не використовувати бази даних;
* логіка має бути реалізована в одному файлі (для базового рівня);
* не використовувати ООП (достатньо базового Python);
* дані вводяться вручну (без файлів на мінімальному рівні).
